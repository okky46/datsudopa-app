# CLAUDE.md — 脱ドパ 開発ガイド

> **`MVP_REQUIREMENTS.md` を現行仕様の最優先情報源とする。**
> 既存コード、README、コメント、過去設計、本ファイルの記述と矛盾する場合は
> `MVP_REQUIREMENTS.md` を優先し、不要な旧実装を残さない。

Expo + React Native + TypeScript（expo-router）のMVP。このファイルは実装・UI修正時に「AIっぽくない・洗練された・親しみやすい脱ドパらしさ」と、MVPのシステム方針を維持するための基準。

## プロダクトコンセプト

- 「ショートの真逆を、みんなで3分だけやる。」毎日22:00 JSTの通知から3分以内に公式レイドへ参加し、知らない人たちと3分間何も起きない映像を見る。
- 紹介文は「ショート中毒者が、毎晩同じ時間に集まって、3分間何も起きない映像を見るアプリ。」
- トーンは「静かなウェルネスアプリ × 少しふざけた世界観」。UIは清潔で柔らかく、コピーは「ドパガキ」「虚無」など脱力系のユーモアを混ぜる。説教くさくしない。
- 「ドパガキ度」は医学的指標ではなく、アプリ内のエンタメ指標。

## システム原則（MVP_REQUIREMENTS.md より）

- **ローカルファースト**: レイド判定・累計脱ドパ時間・ドパガキ度・履歴・連続日数は端末内で完結する。外部通信なしでも公式レイドが成立すること。
- **DB通信最小化**: 公式レイド1回あたりのSupabase通信は最大3回（開始時保存・終了時更新・同行者名3件取得）。Realtime・Presence・ポーリング・レイド中の接続維持は禁止。
- **通信失敗に強く**: Supabase/manifest/動画取得の失敗でアプリを止めない。失敗分は端末内キューへ保存して後から同期する。通信完了を画面遷移やローカル記録の条件にしない。
- **レイド中の情報量を増やさない**: レイド中は映像・残り時間・中断操作のみ。点・名前・人数・広告・チャットは表示しない。
- **架空の参加者を出さない**: 疑似人数・疑似同行者・日付シードの架空ユーザー生成は禁止。同行者名はSupabaseの実データ最大3件のみ。取得できなければ代替の一文を出す。
- **二重加算防止**: すべての視聴セッションは一意な `session_id` を持ち、累計脱ドパ時間・ドパガキ度へ二重反映しない。
- **未使用コードを残さない**: 仕様変更で不要になった型・定数・サービス・コピー・コメントは削除する。
- **新規依存関係は必要性を確認**: 追加時は `npx expo install` でSDK互換版を選ぶ。
- **プラン名で分岐しない**: 課金系の分岐は `FeatureGate`（機能権限）と広告モード（normal / hidden / heavy）で判定する。実課金・偽の購入ボタンは作らない。

## UIデザイン原則

1. 静かで軽い高級感。派手な装飾より余白と階層で見せる。
2. 光る演出（グラデーション）は「ここぞ」に限定: スコアバー、再生ボタン程度。画面縁の光る枠・虹色セレブレーション・パチンコ風演出は使わない。
3. アイコンは絵文字やアイコンフォントではなく、View で手描きした小さなグリフ（ClockGlyph, LockGlyph 等）。この手作り感を維持する。
4. コピーは `src/constants/copy.ts` に集約。画面に説明文を増やさず、世界観の一文で語る。
5. 新しい色・数値をハードコードしない。必ず `src/constants/theme.ts` のトークンを使う。

## 色・余白・角丸・影・タイポグラフィ

すべて `src/constants/theme.ts` が唯一の情報源。1b「上質・ダーク」の方針:

- **色**: 深いインク背景（#14161B）＋クリーム文字（#ECE7DD）＋シャンパンゴールド（#C9A96A）のアクセント。primary はゴールド、達成系はセージグリーン（#9FB79C）、警告寄りはダスティローズ（#CE9A8E）。カード/サーフェスは白の極薄半透明。モーダルなど不透明が必要な箇所だけ `colors.cardOpaque`。純黒・原色のネオンは使わない。
- **余白**: `spacing`（6/10/16/22/32/44）。画面の左右パディングは `spacing.lg`。
- **角丸**: `radius`（12/18/24/30/pill）。カードは lg〜xl、ボタンは pill。
- **影**: `shadows.soft` と `shadows.hero` の2種のみ。
- **タイポグラフィ**: 丸ゴシック `Zen Maru Gothic`。独自 Text スタイルは `zenMaru(weight)` または `fontFamily.{regular|medium|bold|black}` を fontWeight とセットで指定（fontWeight だけでは反映されない）。
- **グラデーション**: `gradientBar`（スコアバー）/ `gradientPlay`（CTA）/ `gradientWash`（主役カード）を `SoftGradient` 経由で使う。
- **一点物のアルファ値**: 特定カードだけの微妙なボーダー/影の透明度は現場リテラル可。ただし色相はトークン由来にする。

## 画面別ルール

### オンボーディング `app/onboarding.tsx`
- 5画面: 問題提起 → ショート利用時間の選択 → 損失時間の可視化 → 22時レイド説明＋公開ユーザーネーム設定 → 初回3分ロング開始。
- ユーザーネームは自動生成3候補＋再生成＋自由入力。バリデーションは `src/utils/username.ts`。
- 損失表示は月間・年間まで。「人生で◯年分」など前提不明確な長期換算は使わない。
- 広告は表示しない。

### ホーム `app/(tabs)/index.tsx`
- 表示順: 次回レイド（カウントダウン/開始/追い脱ドパ） → 累計脱ドパ時間（主指標） → ドパガキ度 → 連続日数 → 今週の履歴 → AdBanner。
- 参加予定・予定人数・参加人数は表示しない。
- 22:00:00〜22:02:59 のみ公式レイド開始可能。22:03以降は「追い脱ドパ」導線。

### ロング `app/(tabs)/long.tsx`
- 無料ユーザーには「今日の1本」だけを提示。動画選択UIは出さない（将来のプレミアム権限 `video_selection` で解放）。
- 視聴時間は 3/10/30/60分プリセットから選択。
- 画面最下部にのみ AdBanner。視聴中は広告なし。

### レイド/視聴 `app/raid/active.tsx`
- レイド中の表示は映像・残り時間・中断操作のみ。スキップ/倍速の偽ボタン、ヒント文言、点・人数は置かない。
- 中断ボタン・background/inactive・継続不能な再生エラーは途中離脱扱い。外部通信障害だけでは失敗にしない。

### リザルト `app/raid/result.tsx`
- 完走/途中離脱、今回の視聴時間、増えた脱ドパ時間、累計、ドパガキ度と増減、連続日数、同行者名（実在・最大3人）、共有、次回22時案内。
- 広告なし。派手なセレブレーションなし。責める表現なし。
- 同行者が取得できない場合は代替の一文（架空名は出さない）。

### 設定 `app/(tabs)/menu.tsx`
- 公開ユーザーネーム、通知ON/OFF、毎日22時のレイド説明、動画キャッシュ削除、オンボーディング再表示、プレミアム/ヘビーモード予定、プライバシーポリシー・利用規約、データ削除、AdBanner。
- レイド時刻の変更UIは置かない（毎日22:00固定）。
- 設定変更は即 `StorageService.saveSettings` + `NotificationService.scheduleDailyRaid`。

## 避けるべきAIっぽいUI・コピー

- 紫→青のビビッドグラデーション、ネオン、強いグラスモーフィズム。
- ゴールド以外の彩度の高いアクセントカラーの追加。基調はインク×ゴールド×クリーム。
- 絵文字アイコンの多用、アイコンライブラリの安易な導入。
- 「〜しましょう!」調の説明文、説明過多のサブテキスト、ラベルの英語化。
- カード内カードの過剰ネスト、ダッシュボード化。
- 「耐える」中心の説明、「儀式」、医学的効果の断言、実測していないドーパミン量、説教口調。

## 守るべき既存機能・構造

- レイドフロー: 22:00通知 → 3分以内の開始判定 → 全画面再生 → AppState失敗判定 → ローカル保存 → 非同期Supabase同期 → リザルト。
- `StorageService` 経由のAsyncStorageアクセス（キーを散在させない）。
- `NotificationService.scheduleDailyRaid` の再スケジュール（ホームフォーカス時・設定変更時）。
- AdBanner の配置（ホーム/ロング視聴前/設定の各最下部のみ）と Expo Go でのplaceholderフォールバック。
- 動画取得は `VideoDeliveryService` に集約。remote manifest → cached manifest → bundled manifest → 同梱映像フォールバックの順。
- オンボーディング完了前のリダイレクト。

## 実装時の注意

- StyleSheet.create + theme トークンで書く。inline の色コード・マジックナンバー禁止。
- 新規UI部品はまず `src/components/ui/` の再利用を検討。無ければ同じ流儀で追加。
- コピー変更・追加は `src/constants/copy.ts`。数値パラメータ（ドパガキ度の増減等）は `src/constants/` の定数へ。
- Expo Go で動く範囲を壊さない（ネイティブ専用APIはフォールバックを用意）。
- 変更後は `npm run typecheck` を通す。

### 依存関係（npm）のバージョン揃え

Expo SDK が指定する React / React Native 系の版を崩さない:

- `react` / `react-dom` / `react-native` は Expo SDK の推奨版に正確にピンする。片方だけ上げない。
- Expo 関連パッケージの追加・更新は `npx expo install <pkg>` を使う。
- `--force` / `--legacy-peer-deps` で握りつぶさない。まずピン揃えで直す。
